# Using Storybook

Storybook is the canonical component reference for Kapwa.

Use Storybook when you need:

- component usage examples
- prop-level guidance
- visual states and variants
- implementation-oriented examples

Use this docs site for the bigger picture: why Kapwa exists, how it is structured, and how teams should adopt or contribute to it. Use Storybook when you need to inspect the components themselves.

<a href="/storybook/" target="_blank" rel="noreferrer">Open Storybook</a>

## Pair Storybook With These Docs

- Start with [Adoption](/guides/adoption) if you are new to Kapwa.
- Use [Installation](/guides/installation) when you are ready to integrate the package.
- Use [Architecture](/guides/architecture) when you need repo structure or build context.
- Use [Contributing](/guides/contributing) when you are changing Kapwa itself.
