# Using Storybook

Storybook is the canonical component reference for Kapwa.

Use Storybook for:

- component usage examples
- prop-level guidance
- visual states and variants
- implementation-oriented examples

Use this docs site for the bigger picture: why Kapwa exists, how it is structured, and how teams should adopt it. Use Storybook when you need to inspect the components themselves.
