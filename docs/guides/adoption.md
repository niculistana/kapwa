# Adoption

Kapwa is meant to be easy to approach, whether you are evaluating it for a product team or preparing to contribute to the system itself.

If you are new to the project, this is the fastest path through the site:

1. Read [Why Kapwa](/guides/why-kapwa) to understand what the design system is trying to solve.
2. Open <a href="/storybook/" target="_blank" rel="noreferrer">Storybook</a> to inspect components, props, and visual states.
3. Follow [Installation](/guides/installation) to add Kapwa to your app.
4. Read [Architecture](/guides/architecture) if you need to understand how the library, docs, and Storybook fit together.
5. Use [Contributing](/guides/contributing) if you plan to change or improve Kapwa itself.

## How To Navigate Kapwa

Kapwa has two main surfaces:

- this docs site explains the system, its purpose, and the technical guidance behind it
- Storybook is the component reference for implementation details

Use this docs site when you need:

- installation steps
- architecture and build context
- design principles and project direction
- contributor workflow guidance

Use Storybook when you need:

- component examples
- prop-level behavior
- states and variants
- implementation-oriented reference while building a feature

## Adoption Checklist

For teams adopting Kapwa without contributing to the library itself:

- confirm Kapwa is the right fit through [Why Kapwa](/guides/why-kapwa)
- inspect the available components in <a href="/storybook/" target="_blank" rel="noreferrer">Storybook</a>
- install the package through [Installation](/guides/installation)
- use this site for architecture and rationale when questions come up

## Contributor Starting Point

For engineers contributing to Kapwa itself:

- set up the repo with [Contributing](/guides/contributing)
- use [Architecture](/guides/architecture) to understand where to make changes
- use [Using Storybook](/guides/using-storybook) to review component behavior while you work
