# Why Kapwa

Kapwa is built to help designers and developers work from the same foundation instead of solving the same problems again and again in separate ways.

It gives teams a shared language for interface design, a component reference they can actually use in implementation, and a technical setup that fits how modern frontend teams already work.

- ⚒️ Aligned with the tools familiar to many developers that allows the design system to live inside a workflow that teams already understand
- 🤗 Kapwa is shaped by care. Designers and developers contribute through collective effort, including UX research to provide clearer, faster, and more humane experiences.

* 🇵🇭 It's already being used across BetterGovPH work efforts, including LGU-scoped portals (see <a href="https://lgu.bettergov.ph/" target="_blank" rel="noreferrer">BetterLGU Directory</a>)

## Why the name matters

Kapwa is a Filipino value often associated with shared identity, togetherness, and recognizing others as fellow human beings rather than distant users or transactions.

That idea matters in digital public service. Government systems should not feel cold, punishing, or needlessly difficult. They should help people move forward with dignity.
